Getting Started

We recommend you start by using the boilerplate. There is two ways to do this, either download the boilerplate from github or install the cli tool and execute the init method.
If you wish to just use the componentjs framework you can download that from github. componentjs on github

Download the boilerplate

Goto the boilerplate on github and download the zip and unzip it into your project folder (unzip on the root) then run the install command.

npm install

Install Componentjs using the cli tool

You can use the cli tool to grab and install the componentjs boilerplate for you. You need to install the dev-cli tool globally first.

npm install -g https://github.com/fahimc/dev-cli.git

Goto you project folder (it can be a git repo) the run the following command to download and install the componentjs boilerplate

dev-cli init componentjs

Running the Proejct

to run the dev server you need to run the following command

npm run dev

Create Component

Components need to have a consistent folder structure and specific componentjs tags etc. There is a simple npm script to help you create components. Use the following command:


      npm run create

You will be asked to provide your component name. this will be what is used to created the folder structure and this will be the component name. As an example we will create a component called 'main-carousel'.


      npm run create
      
      What is the name of your component?  
      main-carousel

      src/component/main-carousel/src/component.js created
      src/component/main-carousel/template/template.html created
      src/component/main-carousel/style/style.scss created

As you can see a new component folder was created and the js, scss and the html file has been created for you.

Let's look at the template file


      <div data-component="main-carousel">
      </div>

The Component template has been generate.

Let's look at the js file.


      export default new Component({
  		name: 'main-carousel',
  		data: { },
  		methods: {
    	mounted(){ },
    	unmounted(){ },
    	updated(){ }
  		}
	});

The component instance has been setup for you and is bound to the 'main-carousel' element tag.

Every time you add 

<component-main-carousel></component-main-carousel>
to the html or to other components this will become an instance of main-carousel and the js code will bind to it.

Component instnace has two object which you can, data and methods.

data{}
This is where you can set variables and in the example above I have created a variable called 'message'. This will be added to you component and you can access it by calling this.message. This will also have one-way binding so when the value changes the updated() method gets called on your component.

methods{}
This is where you can add methods for your component. There are methods which are used by this framework such as mounted(), unmounted() and updated(). You can execute a method by calling it like this this.getMessage()

Development

The folder strucuture of the components and in src are important and you should ensure you development in the same format so the gulp tasks understand where to look for files.

Component templating

In componentjs you have to specify a template element so you don't have to write the entire html for the component everytime you want to use it. This boilerplate has a html replacement task which will replace all 

<component-[componentName]></component-[componentName]>
tags with this component template.

Example 

<component-app></component-app>
will be replace with the template found in src/component/app/template/template.html 

<div data-component="app">
	<component-navigation></component-navigation>
	<div data-router id="main-router">
		<section data-route="/">
			<component-home class="active"></component-home>
			<component-footer></component-footer>
		</section>
		<section data-route="/about">
			<component-about></component-about>
			<component-footer></component-footer>
		</section>
	</div>
</div>

This task can handle nested tags to as shown above. You can also add attributes which will get added to the element replacing the component tag.
