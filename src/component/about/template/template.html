About component.js

This is a mini-framework to give you the ability to build in a modular way by allowing you to build components just like in frameworks such as Vuejs, angular js and react.

Why choose this framework?

This is not like other frameworks, you can bolt this framework on to existing projects and it is designed to have a small footprint for projects with limited file sizes or projects which need to be small in size.

Also, if you have an existing project and you want to slowly migrate it into a component-based architecture then this framework will allow you to do so.

The concept was created when there was a project that had a 150kb file size and there was a need for a component-based architecture.

This framework allows you to take control over what you feel is necessary for the project as component.js only provides the bare minimum needed for building modular.

The boilerplate contains a routers, tag templating and a component creation task which adds the basi for a full-featured development process. Using this you can rapidly build projects.

What's different about this framework?

Apart from its file size, it works with static html. It doesn't render the html at runtime but works with precompiled html. You use data attributes to bind the html template to the component instance. You can create your own libraries and tools to make it suit your needs. It's also much better for SEO since it works with precompiled html.

What projects is the idea for?

This framework is designed for small to large-scale projects. Some examples of projects that this framework is suitable for is listed below:

  • SEO friendly sites
  • Backend (like sitecore) based sites using a modular frontend
  • Banner development
  • SPA's (single page applications)
